Create `src/dataset/exporter.py`:

```python
import io
import json
import zipfile
from src.dataset.models import EvaluationData


def export_dataset_to_zip(data: EvaluationData) -> io.BytesIO:
    """
    Export evaluation dataset to a ZIP file matching 数据集提交指南.md format.

    ZIP structure:
      evaluation_data.json   (UTF-8 JSON with {"test_cases": [...]})
      attachments/           (empty directory — required for chat:text type)

    Returns:
        BytesIO buffer containing the ZIP file, seeked to position 0.
    """
    buffer = io.BytesIO()

    with zipfile.ZipFile(buffer, "w", zipfile.ZIP_DEFLATED) as zf:
        # Serialize test cases — model_dump() excludes fields with exclude=True
        # (source_document, source_chunk_id are excluded automatically)
        json_content = json.dumps(
            {"test_cases": [tc.model_dump() for tc in data.test_cases]},
            ensure_ascii=False,
            indent=2,
        )
        zf.writestr("evaluation_data.json", json_content.encode("utf-8"))

        # Add empty attachments/ directory (required by spec, empty for chat:text)
        zf.mkdir("attachments")

    buffer.seek(0)
    return buffer
```

Key implementation details:
- Use `json.dumps` with `ensure_ascii=False` to preserve Chinese characters in UTF-8
- Use `indent=2` for human-readable JSON
- Use `zf.mkdir("attachments")` to create an empty directory entry in the ZIP (Python 3.11+ zipfile feature). If `mkdir` is not available, use `zf.writestr(zipfile.ZipInfo("attachments/"), "")` as fallback.
- `model_dump()` on TestCase automatically excludes `source_document` and `source_chunk_id` because they have `exclude=True`
- Buffer is seeked to 0 before return so it can be read immediately

Create `src/api/routes/dataset.py`:

```python
import logging
from fastapi import APIRouter, HTTPException
from fastapi.responses import StreamingResponse

from src.dataset.generator import generate_qa_pairs
from src.dataset.exporter import export_dataset_to_zip
from src.core.vectorstore import get_vectorstore

logger = logging.getLogger(__name__)

router = APIRouter(prefix="/dataset", tags=["dataset"])


@router.post("/generate")
async def generate_dataset():
    """
    Generate evaluation Q&A pairs from all indexed documents and return as ZIP.

    Triggers Q&A pair generation for all document chunks in ChromaDB,
    packages the result as a ZIP file conforming to 数据集提交指南.md,
    and returns it as a downloadable file.

    Returns:
        StreamingResponse with application/zip content type
    """
    try:
        vectorstore = get_vectorstore()

        # Generate Q&A pairs from all indexed documents
        evaluation_data = generate_qa_pairs(vectorstore)

        if not evaluation_data.test_cases:
            raise HTTPException(
                status_code=404,
                detail="知识库中没有已索引的文档，无法生成评测数据集。请先上传文档。"
            )

        logger.info(f"Generated {len(evaluation_data.test_cases)} Q&A pairs")

        # Export to ZIP
        zip_buffer = export_dataset_to_zip(evaluation_data)

        return StreamingResponse(
            zip_buffer,
            media_type="application/zip",
            headers={
                "Content-Disposition": "attachment; filename=evaluation_dataset.zip"
            },
        )

    except HTTPException:
        raise
    except Exception as e:
        logger.error(f"Dataset generation failed: {e}", exc_info=True)
        raise HTTPException(
            status_code=500,
            detail=f"数据集生成失败: {str(e)}"
        )
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client HTTP request -> POST /dataset/generate | Unauthenticated endpoint that triggers potentially expensive LLM generation |
| EvaluationData -> ZIP file | Internal data serialized to a downloadable file — data exfiltration by design |
| ZIP file -> client response | Binary file returned to caller — must not contain path traversal entries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Denial of Service | POST /dataset/generate | mitigate | Endpoint triggers LLM calls for all indexed chunks — could be expensive. The endpoint has no request body parameters that amplify cost. Rate limiting should be applied at the infrastructure level (e.g., nginx or FastAPI middleware). Log generation counts. The DashScope API itself has rate limits that provide a natural ceiling. |
| T-03-06 | Elevation of Privilege | ZIP path traversal | mitigate | All filenames written to ZIP are hardcoded constants: `"evaluation_data.json"` and `"attachments/"`. No user input influences ZIP entry names. Use `zipfile.ZipFile` write mode only (no extraction). This eliminates path traversal risk entirely. |
| T-03-07 | Information Disclosure | evaluation_data.json content | accept | The ZIP intentionally contains answers derived from enterprise documents. This is the feature's purpose. Access control to the endpoint should be handled at the API gateway / auth middleware level (not in scope for this phase — single-tenant internal deployment). |
| T-03-08 | Spoofing | Unauthenticated POST /dataset/generate | accept | The entire API is designed for single-tenant internal deployment behind enterprise firewall. Authentication is handled by `Authorization: Bearer $API_KEY` at the API layer (from Phase 1/2). No additional auth needed for this endpoint specifically. |
| T-03-09 | Tampering | ZIP output integrity | mitigate | ZIP is generated in-memory using Python's `zipfile` module with `ZIP_DEFLATED` compression. The buffer is created fresh per request, never reused or cached. No file system write occurs — the ZIP exists only in memory and is streamed directly to the response. |
</threat_model>
