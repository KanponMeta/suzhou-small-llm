**app/config.py:**
```python
from pydantic_settings import BaseSettings
from functools import lru_cache


class Settings(BaseSettings):
    """Application settings loaded from environment variables."""

    # DashScope
    dashscope_api_key: str = ""

    # LLM Models
    llm_model_name: str = "qwen-plus"
    embedding_model_name: str = "text-embedding-v3"
    embedding_dimension: int = 1024

    # ChromaDB
    chroma_host: str = "chromadb"
    chroma_port: int = 8000
    chroma_collection_name: str = "enterprise_kb"

    # API
    api_host: str = "0.0.0.0"
    api_port: int = 8000

    # Upload
    upload_dir: str = "/app/uploads"

    model_config = {
        "env_file": ".env",
        "env_file_encoding": "utf-8",
    }


@lru_cache()
def get_settings() -> Settings:
    return Settings()
```

Note: This uses `pydantic-settings` which is bundled with `pydantic[settings]`. Add `pydantic-settings` to requirements.txt if the import fails. Check: `python -c "from pydantic_settings import BaseSettings"`.

**app/main.py:**
```python
from fastapi import FastAPI
from app.config import get_settings

settings = get_settings()

app = FastAPI(
    title="Enterprise Knowledge Base RAG",
    description="Enterprise document knowledge base with RAG-powered Q&A",
    version="0.1.0",
)


@app.get("/")
async def root():
    return {"message": "Enterprise Knowledge Base RAG System", "version": "0.1.0"}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host -> Docker network | Docker Compose internal network isolates services |
| .env file -> container | API keys loaded from .env into container environment |
| External -> API port 8000 | FastAPI exposed on port 8000, accepts HTTP requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation |
|-----------|----------|-----------|----------|-------------|------------|
| T-01-01 | Information Disclosure | .env file containing DASHSCOPE_API_KEY | high | mitigate | .env is in .dockerignore and .gitignore; .env.example has placeholder only; never commit real keys |
| T-01-02 | Information Disclosure | Docker image layers may cache secrets | medium | mitigate | requirements.txt copied before app code (layer caching); no COPY of .env in Dockerfile; secrets injected via env_file at runtime |
| T-01-03 | Denial of Service | ChromaDB port exposed on host (8100) | low | accept | Internal enterprise deployment; ChromaDB healthcheck endpoint is read-only |
| T-01-04 | Elevation of Privilege | Container running as root | medium | accept | Single-tenant internal deployment; can add non-root user in hardening phase |
